# 短剧脚本与分镜生成 Skill

## 功能描述
根据已存在的大纲和参考素材，自动生成每集的剧情脚本与分镜脚本，为后续的 AI 图像/视频生成提供详细的镜头描述。

## 输入文件
- `workspace目录/duanju/<剧名>/大纲.md`：剧集整体大纲，包含剧情走向、集数、人物设定等
- `workspace目录/duanju/<剧名>/参考素材.md`：重点人物、重点场景的参考描述与图片信息
- `skill.md`：全局配置（ComfyUI 远程调用参数、API 地址、参数模板等）

## 输出文件
- `workspace目录/duanju/<剧名>/<集数>_台词.md`：每集带时间戳的台词脚本（用于短视频、字幕与 TTS 对齐）
- `workspace目录/duanju/<剧名>/<集数>_脚本.md`：每集的剧情脚本
- `workspace目录/duanju/<剧名>/<集数>_分镜.md`：每集的分镜脚本

---

## 流程说明

### 第一步：读取上下文
1. 读取 `大纲.md`，提取：
   - 剧名、集数
   - 每集剧情概要、起承转合
   - 人物列表及其性格、外貌特征
   - 重点场景列表
2. 读取 `参考素材.md`，提取：
   - 重点人物的标准描述（用于分镜中引用）
   - 重点场景的标准描述（用于分镜中引用）
3. 读取 `skill.md`，获取 ComfyUI 调用相关配置（后续步骤使用，本步骤仅作为信息收集）

### 第二步：生成每集剧情脚本 `<集数>_脚本.md`
针对每一集，生成内容包含：
- **集标题**与集序号
- **场景列表**：本集出现的场景（按时间顺序）
- **出场人物**：本集出场的人物
- **剧情正文**：包含对白、动作描写、心理活动、镜头氛围
- **节奏标注**：高潮点 / 反转点 / 情绪转折

格式示例：
```
# 第01集 <标题>

## 场景
1. 街道 - 白天
2. 咖啡馆内 - 白天
...

## 出场人物
- 主角A
- 配角B
...

## 剧情正文
（详细对白与动作描写……）

## 节奏标注
- 开场引入：...
- 高潮点：...
```

### 第三步：生成每集分镜脚本 `<集数>_分镜.md`
针对每一集脚本，拆解为多个镜头。每个镜头包含：
- **镜头编号**：如 Shot 01、Shot 02 …
- **景别**：远景 / 全景 / 中景 / 近景 / 特写
- **运镜**：推 / 拉 / 摇 / 移 / 跟 / 固定
- **画面描述**：人物动作、表情、构图
- **引用标注**：
  - 重点人物 → 引用 `参考素材.md` 中的标准人物描述标签，如 `@人物:主角A`
  - 重点场景 → 引用 `参考素材.md` 中的标准场景描述标签，如 `@场景:咖啡馆`
- **对白**：由本 Shot 内对白/旁白的音轨时长计算得出（音轨时长 = 文本字数 ÷ 该角色语速，语速取自 `参考素材.md`，标注方式参考 `duanju-scene-prompt.md`），与 `台词.md` 中对应条目的时间码区间完全一致镜头的文字内容
- **时长建议**：秒数

格式示例：
```
# 第01集 分镜

## Shot 01
- 景别：远景
- 运镜：固定
- 画面：@场景:城市街道 日景，@人物:主角A 走在人群中
- 对白：——
- 时长：3s

## Shot 02
- 景别：近景
- 运镜：推
- 画

### 第三步.b：生成每集台词脚本 `<集数>_台词.md`
针对每一集分镜，**同步**生成带时间戳的台词脚本，用于短视频画面、字幕与 TTS 配音的三方对齐。每个条目包含：
- **时间码**：精确起止时间（格式 `HH:MM:SS - HH:MM:SS`），累计时长与对应 `Shot` 的「时长」字段一致
- **镜头引用**：对应的 `Shot` 编号（如 `Shot 01`）
- **角色**：说话人，引用分镜中 `@人物:` 标签；旁白写「旁白」
- **类型**：对白 / 旁白
- **文本**：从 `参考素材.md` 中读取对白/旁白文本
- **音色**：从 `参考素材.md` 中该人物的音色记录读取；旁白音色从 `参考素材.md` 中的「全剧旁白音色」记录读取
- **语速**：从 `参考素材.md` 中该人物的语速记录读取；不再使用全局默认值，旁白若无指定则沿用 `参考素材.md` 中「旁白」角色的语速（建议 3-5 字/秒，留出停顿）
- **字幕**：可由本表直接生成 SRT/ASS 字幕文件
```

- **音轨时长计算**：每条对白/旁白的音轨时长 = 文本字数 ÷ 该角色语速（向上取整，最小 1s）；句末标点自动 +0.3s，对白/旁白切换自动 +0.2s

### 第四步：循环直到全部集数完成
对大纲中的所有集数依次执行第二步、第三步，直到：
- `workspace目录/duanju/<剧名>/<集数>_脚本.md` 全部生成
- `workspace目录/duanju/<剧名>/<集数>_分镜.md` 全部生成

### 第五步：进入后续步骤
所有脚本与分镜生成完成后，进入下一步骤（如调用 ComfyUI 生成图片/视频）。

---

## 关键约束
1. **人物一致性**：分镜中所有重点人物必须通过 `@人物:` 标签引用 `参考素材.md` 的标准描述，确保后续生图一致性。
2. **场景一致性**：分镜中所有重点场景必须通过 `@场景:` 标签引用 `参考素材.md` 的标准描述。
3. **分镜粒度**：每集分镜不少于 N 个镜头（建议 8-15 个，根据剧情长度调整）。
4. **编号规范**：集数使用两位数字（如 `01`、`02`），便于文件排序。
5. **不修改大纲**：本步骤仅读取大纲，不修改其内容。

---

## 异常处理
- 若大纲中集数为空：提示用户补充大纲。
- 若参考素材缺失重点人物/场景描述：在分镜中使用通用描述并标注 `[待补充]`。
- 若某集剧情过长：拆分为 Part A / Part B，分别生成 `_脚本.md` 与 `_分镜.md`。

## 第五步（续）：更新参考素材
在所有脚本与分镜生成完成后，更新 `workspace/duanju/<剧名>/参考素材.md` 文件，追加或更新以下内容：
- **每一集标题**：列出本剧全部集数的标题
- **每一集的 md 文件清单**：记录每一集生成的 `脚本.md` 与 `分镜.md` 的完整文件路径（如 `workspace/duanju/<剧名>/01_脚本.md`、`workspace/duanju/<剧名>/01_分镜.md`），便于后续 ComfyUI 生成时定位调用
- **分镜序号**：每一集对应的分镜编号清单（如 `01_分镜.md` → Shot 01 - Shot 15）
- **需要使用的素材**：每一集分镜中引用的 `@人物:`、`@场景:` 标签汇总，便于后续 ComfyUI 生成时按需调用
